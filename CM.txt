# Yêu cầu: Xây dựng hệ thống comment tích hợp với hệ thống đăng nhập Supabase hiện có, chạy trên Cloudflare Pages

## 1. Tổng quan
Website hiện tại đã sử dụng **Supabase Auth** để quản lý người dùng (đăng ký, đăng nhập, OAuth). Bạn cần xây dựng một hệ thống comment nhẹ, đẹp, hiệu năng cao, chạy trên **Cloudflare Pages**, tận dụng Cloudflare D1 làm database lưu comment và Cloudflare KV để cache. Hệ thống comment phải đồng bộ với session Supabase hiện có: chỉ người dùng đã đăng nhập mới được gửi comment, và comment sẽ gắn với user_id của họ.

Các yêu cầu chi tiết:
- **Tích hợp Supabase Auth sẵn có**: không xây dựng lại auth, chỉ sử dụng JWT hiện tại để xác thực API.
- **Backend**: Cloudflare Pages Functions (TypeScript/JavaScript) + D1 (lưu comment) + KV (cache).
- **Frontend**: Một component comment độc lập, có thể nhúng vào bất kỳ trang tĩnh nào, tự động nhận diện trạng thái đăng nhập từ Supabase client hiện có.
- **Giao diện comment** hiện đại, responsive, hỗ trợ markdown cơ bản, avatar từ Supabase (nếu có) hoặc Gravatar, phân trang (load more).
- **Tối ưu hiệu năng và tiết kiệm quota**:
  - Chỉ gọi API lấy danh sách comment khi bài viết có comment (kiểm tra qua endpoint `/has` cache KV).
  - Cache danh sách comment trong KV (TTL 5 phút) để giảm tải D1.
  - Phân trang để giảm lượng dữ liệu mỗi lần load.
- **Chống spam cơ bản**: rate limiting (KV), honeypot, validation.

## 2. Công nghệ sử dụng
- **Frontend**: HTML/CSS/JS thuần, sử dụng Supabase JS client đã có sẵn trong website để lấy session và token.
- **Backend API**: Cloudflare Pages Functions (TypeScript), D1, KV.
- **Database**: Cloudflare D1 (SQLite) với bảng `comments`.
- **Cache**: Cloudflare KV.
- **Xác thực**: Sử dụng JWT từ Supabase client hiện có, gửi qua header `Authorization: Bearer <token>` khi gọi API ghi comment.

## 3. Database schema (D1)
```sql
CREATE TABLE IF NOT EXISTS comments (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  post_slug TEXT NOT NULL,
  parent_id INTEGER DEFAULT 0, -- hỗ trợ trả lời
  user_id TEXT NOT NULL, -- Supabase user ID
  author_name TEXT NOT NULL, -- lấy từ Supabase user metadata hoặc profiles
  author_email TEXT,
  author_avatar TEXT,
  content TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  status TEXT DEFAULT 'approved', -- nếu cần kiểm duyệt, có thể để 'pending'
  ip TEXT,
  user_agent TEXT
);
CREATE INDEX idx_post_slug ON comments(post_slug);
CREATE INDEX idx_user_id ON comments(user_id);
CREATE INDEX idx_status ON comments(status);
CREATE INDEX idx_created_at ON comments(created_at);
4. API Endpoints (Cloudflare Pages Functions)
Tạo thư mục functions/api/comment với các endpoint:

GET /api/comment/has?postSlug=xxx
Mục đích: Kiểm tra xem bài viết có comment đã duyệt hay không.

Cache: KV với key has:${postSlug}, cập nhật khi có comment mới.

Không yêu cầu token.

GET /api/comment?postSlug=xxx&page=1&limit=20
Mục đích: Lấy danh sách comment đã duyệt của bài viết, phân trang.

Cache: KV với key comments:${postSlug}:page:${page}, TTL 5 phút.

Không yêu cầu token.

POST /api/comment
Mục đích: Tạo comment mới.

Yêu cầu: Header Authorization: Bearer <supabase_jwt>.

Kiểm tra token: giải mã JWT, lấy user_id. Có thể gọi Supabase để lấy thông tin user nếu cần (name, email, avatar) hoặc dùng metadata trong token.

Chống spam: rate limiting (KV theo IP, 5 lần/5 phút), honeypot (field ẩn trong form).

Lưu vào D1 với user_id và thông tin user (lấy từ token hoặc gọi Supabase).

Sau khi lưu:

Xóa cache KV của has:${postSlug}.

Xóa cache của trang comment đầu tiên (hoặc tất cả các trang liên quan, có thể xóa prefix comments:${postSlug}:).

Trả về comment vừa tạo (kèm thông tin user).

OPTIONAL: DELETE /api/comment/:id
Mục đích: Xóa comment (chỉ chủ comment hoặc admin).

Yêu cầu token, kiểm tra quyền.

5. Frontend Component
Tạo script comments.js độc lập, sử dụng Supabase client đã có trong website:

html
<div id="comments-container" data-post-slug="bai-viet-1"></div>
<script src="/comments.js" defer></script>
Script sẽ:

Lấy Supabase client hiện tại từ window.supabase hoặc tạo client mới với URL và key công khai (nếu chưa có).

Kiểm tra trạng thái đăng nhập qua supabase.auth.getSession().

Nếu chưa đăng nhập: hiển thị form với nút "Đăng nhập để bình luận" (dùng supabase auth signIn).

Nếu đã đăng nhập: hiển thị form comment với tên/avatar của người dùng (lấy từ user metadata).

Gọi API /has trước để quyết định có load danh sách comment không.

Load danh sách comment với phân trang (load more), render comment với avatar, markdown.

Gửi comment: lấy token từ session, gửi POST kèm header Authorization.

Cập nhật giao diện sau khi gửi thành công: thêm comment mới vào danh sách, reset form.

Có cơ chế retry khi gọi API lỗi.

Sử dụng localStorage để lưu tạm nội dung comment đang soạn (optional).

6. Tối ưu hiệu năng và tiết kiệm quota
KV cache: như đã nêu ở API.

Tránh gọi API không cần thiết: chỉ gọi /has, nếu false thì không gọi /comment.

Lazy load: Chỉ load comment khi người dùng scroll đến vùng comment (hoặc dùng Intersection Observer).

Phân trang: Giảm tải D1 và network.

Cache client: Có thể cache danh sách comment trong localStorage với timestamp để giảm gọi API khi quay lại trang (optional).

7. Bảo mật
Xác thực token trên server cho POST endpoint.

Parameterized queries trong D1.

Sanitize nội dung comment trước khi lưu và hiển thị (dùng DOMPurify).

Rate limiting (KV) để chống spam.

Honeypot (trường ẩn trong form) để chống bot.

8. Triển khai và hướng dẫn
Cung cấp file wrangler.toml / wrangler.jsonc cấu hình D1 và KV.

Hướng dẫn tạo D1 database, chạy migration.

Hướng dẫn cấu hình biến môi trường cho Supabase URL, anon key (nếu cần trong function) và JWT secret (để verify token).

Hướng dẫn deploy lên Cloudflare Pages.

Cung cấp mã nguồn đầy đủ, chú thích rõ.
